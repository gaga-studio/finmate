# FinMate 발표용 서비스 기획서 v1.0

작성일: 2026-06-30  
발표 시간: 7분  
용도: 하나 청년 금융인재 양성 프로젝트 최종성과발표회  
문서 목적: v1.0의 발표 메시지를 7분 발표에 맞게 압축하고, 문제 근거와 안전 표현을 보강한다.

## 1. 한 줄 정의

FinMate는 합성 마이데이터 기반 또래 페르소나와 내 금융 상태를 비교하고, 가장 큰 격차를 3개월 시뮬레이션과 오늘의 미션으로 바꾸는 청년 금융 온보딩 서비스다.

핵심 문장:

> 또래 비교 → 3개월 시뮬레이션 → 오늘의 미션 → 개인정보 안전장치

발표 데모는 실제 개인정보나 실제 마이데이터를 사용하지 않는다. `MOCK_MYDATA`와 `SYNTHETIC_PERSONA` 기반의 합성/mock 데이터로 시연하고, 향후 실제 서비스에서는 `동의 기반 가명·마스킹 또래 금융 사례 카드`로 확장한다.

## 2. 7분 발표에서 남길 메시지

| 순서 | 메시지 | 발표 문장 |
| --- | --- | --- |
| 1 | 문제 | 청년에게 부족한 건 금융 지식 자체보다 오늘 움직이게 하는 행동 계기다. |
| 2 | 인사이트 | SNS 비교는 소비와 경험을 자극하지만, 저축·비상금·금융 루틴은 잘 보이지 않는다. |
| 3 | 해결 | FinMate는 비교 자극을 금융 행동으로 바꾼다. |
| 4 | 데모 | 30초 진단에서 시작해 또래 사례, 격차, 3개월 변화, 오늘의 미션까지 이어진다. |
| 5 | 안전 | 합성 데이터, 동의, 마스킹, 공개 미리보기, 철회, audit log를 전제로 한다. |
| 6 | 확장 | Birthday Wish Money는 P0가 아니라 Social Impact 확장 사례다. |
| 7 | 하나금융 | 하나금융은 판매자가 아니라 신뢰 가능한 정보, 교육, 리워드, 장기 온보딩 기반이다. |

## 3. 7분용 10장 구성

총 시간은 420초로 고정한다. 14장 구성은 내부 제작 참고안으로만 남기고, 실제 발표자료는 아래 10장 기준으로 만든다.

| 슬라이드 | 제목 | 핵심 내용 | 예상 시간 |
| --- | --- | --- | ---: |
| 1 | FinMate: 비교를 금융 행동으로 | 제품명, 한 줄 정의, 핵심 문장 | 25초 |
| 2 | 문제: 청년에게 부족한 건 지식보다 행동 계기 | 알아도 미루는 금융 행동 문제 | 45초 |
| 3 | 근거: 행동을 막는 이유를 확인한다 | 인터뷰/설문/앱 비교/시장자료 입력칸 | 45초 |
| 4 | 해결 구조: 또래 사례 → 격차 → 시뮬레이션 → 미션 | 서비스 핵심 흐름도 | 45초 |
| 5 | 데모 1: 30초 진단과 홈 | ONB-02, mock 동의, 개인정보 동의, HOME-01 | 45초 |
| 6 | 데모 2: 또래 사례와 mainGap | EXP-03, EXP-04, `peer-portfolio-023`, `EMERGENCY_FUND` | 50초 |
| 7 | 데모 3: 3개월 변화와 오늘의 미션 | SIM-01, MIS-01, 월 10만 원 자동이체 미션 | 50초 |
| 8 | 안전장치: 공개 미리보기와 철회 | SET-01, `own-portfolio-001`, 철회 후 410 | 45초 |
| 9 | Social Impact 확장: 생일 위시머니 | P2 확장, 생활 속 목표형 금융 경험 | 40초 |
| 10 | 하나금융 역할과 마무리 | 신뢰·공식 정보·교육·리워드·장기 온보딩 | 30초 |

합계: 420초

## 4. 문제 근거 1장 템플릿

임의 수치는 넣지 않는다. 발표자료 제작 전 실제 인터뷰/설문/앱 비교/시장자료 중 최소 2개 이상을 채운다.

| 근거 종류 | 최소 수집 가이드 | 채워 넣을 증거 형식 | 현재 상태 |
| --- | --- | --- | --- |
| 청년 인터뷰 | 3명 이상, “금융 행동을 미룬 경험”, “오늘 바로 시작하려면 필요한 계기” 질문 | 직접 발화 2~3개, 공통 키워드, 행동을 막는 이유 | [추가 필요] |
| 간단 설문 | 20명 내외, 미루는 이유/또래 비교 반응/미션 수용도 질문 | 응답 비율, 상위 선택지, 짧은 인사이트 | [추가 필요] |
| 기존 앱 비교 | 금융 앱 3개 이상, 리포트/또래 비교/미션/동의 철회 UI 비교 | O/X 표, “분석 이후 행동 전환” 공백 | [추가 필요] |
| 시장/기관 자료 | 청년 금융교육, 마이데이터, 금융습관 형성 관련 공식 자료 | 출처 2개 이상과 시사점 | [추가 필요] |

발표 문장 예시:

```text
이번 프로젝트의 출발점은 “청년이 금융을 모른다”가 아닙니다.
문제는 알아도 움직일 계기가 약하다는 점입니다.
그래서 FinMate는 지식을 더 설명하기 전에,
내 또래와의 차이를 보여주고 오늘 행동으로 바꾸는 흐름에 집중했습니다.
```

## 5. P0 데모 흐름

```text

→ mock 마이데이터 동의

→ demo accessToken 발급
→ 홈 금융 요약
→ 또래 사례 카드
→ 1:1 비교와 가장 큰 격차
→ 3개월 시뮬레이션
→ 오늘의 미션
→ 개인정보 공개 미리보기/철회
```

| 단계 | 화면 | 발표 포인트 | 개발 연결 |

| 4 | ONB-02 | 익명 공개와 친구 공유 기본값을 별도로 동의받는다. | `POST /api/privacy/consents` |

## 6. 7분 발표 원고

```text
FinMate는 청년이 금융을 몰라서가 아니라,
지금 행동해야 할 계기를 체감하지 못한다는 문제에서 시작했습니다.
우리는 친구의 소비와 여행은 매일 보지만,
친구가 어떻게 비상금을 만들고 저축 루틴을 유지하는지는 잘 보지 못합니다.

FinMate는 이 비교 자극을 금융 행동으로 바꿉니다.
30초 진단으로 사용자의 상태와 목표를 받고,
발표용 합성 마이데이터와 개인정보 공개 범위를 동의받은 뒤
홈에서 내 금융 요약과 또래 사례 진입점을 보여줍니다.

또래 사례는 평균값만 보여주는 것이 아니라
비슷한 조건의 합성 페르소나가 어떤 루틴을 가지고 있는지 카드처럼 보여줍니다.
내 데이터와 비교하면 가장 큰 격차인 mainGap이 나오고,
이번 데모에서는 비상금 준비율이 핵심 격차입니다.

그 다음 월 10만 원을 더 모으면 3개월 뒤 비상금 준비율이
0.4개월에서 0.7개월로 바뀐다는 시뮬레이션을 보여줍니다.
마지막으로 오늘의 미션은
이번 달 비상금 자동이체 10만 원 설정하기처럼 작고 실행 가능한 행동으로 제안됩니다.

여기서 AI는 금융 판단을 대신하지 않습니다.
P0는 rule-based 코치형 추천으로 안정적으로 완주하고,
LLM은 이후 설명을 자연스럽게 만드는 선택 고도화로 둡니다.

마지막으로 안전장치를 보여줍니다.
데모는 실제 마이데이터가 아니라 합성 데이터 기반이며,
향후 실제 서비스에서도 공개 미리보기, 마스킹, 철회, audit log를 전제로 합니다.

FinMate는 이후 Social Impact 확장으로 생일 위시머니를 붙일 수 있습니다.
이 기능은 생일 송금 앱이 아니라,
친구의 선물을 목표 금액 설정, 소액 적립, 달성 경험으로 바꾸는 생활형 금융 경험입니다.
하나금융은 여기서 상품을 바로 판매하는 주체가 아니라,
공식 정보와 교육 콘텐츠, 리워드, 신뢰 인프라로 청년의 첫 금융 행동을 돕는 역할입니다.
```

## 7. Social Impact 확장 슬라이드

Birthday Wish Money는 P0 데모에 넣지 않는다. 7분 발표에서는 “P0 이후 확장 가치”로 40초 안에 설명한다.

| 관점 | 설명 |
| --- | --- |
| 기능 정의 | 친구의 생일 선물을 단순 소비가 아니라 목표 금액 설정, 소액 적립, 달성 경험으로 바꾸는 생활형 금융 기능 |
| Social Impact 의미 | 청년이 적립식 금융, 위험과 수익, 분산, 기간 같은 기초 개념을 작은 생활 이벤트로 먼저 경험하게 함 |
| 하나금융 연결 | 공식 교육 콘텐츠, 조건 확인, 퀴즈, 리워드와 연결해 상품 가입 압박 없이 금융 이해도를 높임 |
| 구현 우선순위 | P2. P0 데모 안정성을 위해 이번 구현 범위에서는 제외 |

금지 표현:

| 피해야 할 표현 | 권장 표현 |
| --- | --- |
| 생일 펀드로 펀드 투자를 경험 | 생일 위시머니로 목표 금액 설정, 소액 적립, 달성 과정을 경험 |
| 투자상품 가입 유도 | 생활 속 목표형 금융 경험 |
| ESG 기능 | Social Impact 확장 |

## 9. 발표자료 제작 체크리스트

- [ ] 10장 구성으로 420초 안에 리허설한다.
- [ ] 문제 근거 슬라이드에는 임의 수치를 넣지 않는다.
- [ ] ONB-02부터 SET-01까지 데모 흐름이 끊기지 않는다.
- [ ] `mainGap`은 “가장 큰 격차를 하나로 고르는 장치”로만 설명한다.
- [ ] 미션 문구는 월 10만 원 자동이체로 통일한다.
- [ ] Birthday Wish Money는 P2 Social Impact 확장으로만 다룬다.
- [ ] 하나금융은 판매자가 아니라 신뢰·교육·리워드·공식 정보 기반으로 설명한다.

## 10. Q&A 백업

| 질문 | 답변 방향 |
| --- | --- |
| 실제 금융정보를 쓰나요? | 발표 데모는 `MOCK_MYDATA`, `SYNTHETIC_PERSONA` 기반이며 실제 개인정보를 쓰지 않는다. |
| 왜 또래 비교가 필요한가요? | 소비 비교로 작동하던 자극을 저축·비상금·루틴 같은 행동 계기로 바꾸기 위해서다. |
| AI가 금융상품을 추천하나요? | 아니다. P0는 rule-based 행동 미션 추천이며 상품 가입 권유나 투자 자문이 아니다. |
| 개인정보 공개는 위험하지 않나요? | 공개 미리보기, 마스킹, 구간값, 철회, audit log를 전제로 한다. |
| 데이터셋은 어디서 왔나요? | 별도 합성 데이터셋에서 P001과 P003 페르소나를 P0 subset으로 연결했다. P001은 저축률이 낮은 데모 사용자, P003은 더 강한 저축 루틴을 가진 또래 사례로 사용한다. 화면 금액, 시뮬레이션, `similarityScore=0.84`는 발표 흐름에 맞춘 `DEMO_NORMALIZED` 값이며 원본 raw cluster distance가 아니다. |
| 생일 위시머니는 왜 필요한가요? | P0 핵심은 아니지만, 청년이 생활 속에서 목표형 금융 경험을 작게 해보는 Social Impact 확장이다. |
